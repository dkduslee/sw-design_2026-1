## 1. Introduction

본 문서는 Conceptualization Document에 이은 두 번째 단계인 Analysis 단계의 문서로, 본 시스템에 대한 두 번째 문서이다. 이 단계에서는 Actor와 Use case들의 관계를 정리한 Use Case Diagram, 각 Use case를 구체적으로 설명하는 Use case description, 그리고 본 시스템에 필요한 클래스들의 간략한 설명을 담은 Domain analysis를 다룬다.

### 1.1. Executive Summary

교대 근무를 하는 사람들은 매주 바뀌는 근무 스케줄로 인해 알람을 매번 수동으로 재설정해야 하는 불편함을 겪는다. 기존의 알람 앱은 고정된 패턴에 최적화되어 있어 주간·야간·비번·휴무가 반복되는 3교대 근무자에게는 적합하지 않다. 또한 개인 캘린더와 알람이 연동되지 않아 두 앱을 따로 관리해야 하는 번거로움도 있다.

이러한 문제를 해결하기 위해, 근무 스케줄을 한 번 입력하면 알람과 기기 캘린더가 자동으로 연동되는 **My_timer**이라는 앱을 개발하게 되었다. 이 어플을 개발함에 있어서, 사용자는 훨씬 더 편리하게 본인의 스케줄을 관리할 수 있을 것으로 기대한다.

### 1.2. Business Goals

My_timer는 교대 근무자가 근무 유형(주간·야간·비번·휴무)을 날짜별로 등록하면 해당 유형에 맞는 알람을 자동으로 설정하고, 기기 캘린더에도 이벤트를 자동 등록하여 스케줄 관리에 도움을 주기 위해 개발된다. 편의성을 높여 유동적인 스케줄을 가진 많은 이용자들의 업무 효율성을 높이는 것이 목적이다. 스케줄이 수정되면 알람과 캘린더 이벤트도 즉시 갱신되며, 반복 패턴을 설정하면 다음 주 스케줄도 자동으로 설정되도록 하여 사용자 편의성을 높이고자 한다.

## 3. Domain analysis

### 1) ShiftType

근무 유형을 나타내는 클래스이다. 주간(Day), 야간(Night), 비번(Off-duty), 휴무(Holiday) 등 교대 근무의 각 구분 정보를 가지며, 유형별 기본 알람 시간과 캘린더 표시 색상 정보를 포함한다.

### 2) ShiftSchedule

특정 날짜에 할당된 근무 스케줄 데이터 모델 클래스이다. 날짜, 근무 유형, 시작·종료 시간 정보를 저장한다. SQLite DB에 저장되는 기본 단위이다.

### 3) ScheduleManager

스케줄의 등록, 수정, 삭제(CRUD)를 담당하는 클래스이다. SQLite DB(sqflite 패키지)와 직접 통신하며, 데이터 변경 시 SyncService를 호출한다.

### 4) AlarmSetting

특정 근무 일정에 연결된 알람 정보를 저장하는 클래스이다. 알람 시간, 알림 메시지, 반복 여부 등의 설정 데이터를 가진다.

### 5) AlarmManager

OS AlarmManager와 연동하여 알람을 등록하거나 취소하는 클래스이다. flutter_local_notifications 패키지를 사용하며, Android 12 이상의 SCHEDULE_EXACT_ALARM 권한 처리 로직을 포함한다.

### 6) CalendarEvent

기기 캘린더에 등록되는 근무 일정 이벤트의 데이터 모델 클래스이다. 이벤트 제목, 시작·종료 시간, 색상 정보를 포함한다.

### 7) CalendarManager

device_calendar 패키지를 사용하여 기기 캘린더에 이벤트를 읽고 쓰는 클래스이다. 캘린더 접근 권한 요청 처리와 이벤트 생성·수정·삭제를 담당한다.

### 8) SyncService

스케줄 변경 이벤트를 감지하고 AlarmManager와 CalendarManager를 순서대로 호출하여 알람·캘린더 동기화를 조율하는 클래스이다. 동기화 실패 시 롤백 처리를 담당한다.

### 9) NotificationHelper

푸시 알림 메시지를 생성하고 사용자 기기에 알림을 전달하는 클래스이다. 근무 유형에 맞는 메시지 포맷과 진동·소리 설정을 관리한다.

### 10) MainActivity (HomeScreen)

앱의 진입점이자 UI를 구성하는 클래스이다. 하단 네비게이션 탭(캘린더, 알람, 설정)을 관리하며 각 화면 간 이동을 처리한다.

## 4. User Interface prototype

### 1. 메인 화면 (Calendar Screen)

/ 이미지

앱을 실행하면 가장 먼저 보이는 화면이다. 월간 캘린더가 표시되며, 날짜별로 근무 유형에 따른 색상이 적용된다. 날짜를 탭하면 해당 날짜의 근무 유형을 등록하거나 수정할 수 있다. 하단 네비게이션 바에서 캘린더, 알람 목록, 설정 탭으로 이동할 수 있다.

---

### 2. 근무 유형 선택 화면 (Shift Select Dialog)

/ 이미지

캘린더에서 날짜를 탭하면 나타나는 다이얼로그 화면이다. 주간, 야간, 비번, 휴무 중 하나를 선택할 수 있으며, 반복 패턴 설정 버튼도 포함된다. 선택 후 확인을 누르면 스케줄이 저장되고 알람·캘린더 동기화가 자동으로 실행된다.

---

### 3. 알람 목록 화면 (Alarm List Screen)

/ 이미지

등록된 알람 목록을 날짜 순서대로 보여주는 화면이다. 각 알람 항목에는 날짜, 근무 유형, 알람 시간이 표시된다. 항목을 탭하면 해당 알람의 시간을 수동으로 수정할 수 있다.

---

### 4. 알람 시간 수정 화면 (Alarm Edit Screen)

/ 이미지

특정 날짜의 알람 시간을 수동으로 수정하는 화면이다. 시간 선택 피커가 표시되며, 확인을 누르면 AlarmSetting이 갱신되고 OS 알람도 즉시 재등록된다.

---

### 5. 설정 화면 (Settings Screen)

/ 이미지

앱 전반의 설정을 관리하는 화면이다. 아래 항목들을 설정할 수 있다.

- 근무 유형별 기본 알람 시간 (주간, 야간, 비번, 휴무)
- 알람 선행 시간 (15분 / 30분 / 60분 전)
- 알람 소리 및 진동 on/off
- 근무 유형별 캘린더 색상
- 기기 캘린더 연동 on/off

설정값은 SharedPreferences에 저장되며, 변경 즉시 반영된다.

## 5. Glossary

| Term | Description |
|---|---|
| **근무 유형 (ShiftType)** | 주간(Day), 야간(Night), 비번(Off-duty), 휴무(Holiday) 등 교대 근무의 각 구분 |
| **AlarmManager** | Android OS에서 특정 시간에 알람을 실행시키는 OS 수준의 API |
| **CalendarProvider** | Android OS에서 기기 캘린더 데이터를 읽고 쓰는 ContentProvider API |
| **SyncService** | 스케줄 변경을 감지하고 알람·캘린더 동기화를 조율하는 내부 서비스 클래스 |
| **SQLite (sqflite)** | 앱 내 근무 스케줄 데이터를 로컬에 영구 저장하는 경량 데이터베이스 |
| **SharedPreferences** | 알람 선행 시간, 근무 색상 등 앱 설정값을 key-value 형태로 저장하는 로컬 저장소 |
